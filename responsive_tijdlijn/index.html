Geschiedenis van responsive webdesign

1455

Boekdrukkunst

De geschiedenis van de responsive webdesign begint bij de boekdrukkunst. Deze werd rond 1455 ,in de Middeleeuwen, uitgevonden door Johannes Gutenberg. Hij werkte ongeveer 3 jaar aan de Gutenbergbijbel, wat het eerste boek is dat tot stand kwam door de boekdrukkunst. Dit boek zou later in deze tijdlijn voor inspiratie kunnen zorgen, omdat hier al van structuur gebruik werd gemaakt. In de Gutenbergbijbel was op elke pagina een grid van twee kolommen met een vaste breedte en hoogte.

1850

Onderzeekabels

Na de boekdrukkunst zijn we een paar eeuwen verder. In 1850 werden er kabels in de zee tussen de steden Dover en Calais door de broers Brett aangelegd. Deze kabels zorgden ervoor dat er via de telegraaf tussen Engeland en Frankrijk gecommuniceerd kon worden. De telegraafcommunicatie maakte het de gebruikers mogelijk om binnen enkele minuten met elkaar te communiceren. Diezelfde onderzeekabels werden later ook voor telefoonsignalen gebruikt. Als volgt voor het internet.

1989

HTTP

Doordat er nu onderzeekabels zijn aangelegd en is uitgevonden dat wij daarmee kunnen communiceren, werd het ook mogelijk om data te verzenden. Dit werd in 1989 door Tim Berns-Lee ontwikkeld en heet het Hypertext Transfer Protocol (HTTP). Hij maakte het mogelijk om tussen een webclient en webserver data te verzenden en ontvangen. Tegenwoordig vind je de HTTP aan het begin van de URL, wanneer je op het internet gaat. Dit betekent dat de webclient, wat meestal de webbrowser is, toestemming aan de webserver vraagt om HTML bestanden te mogen ontvangen.

1990

World Wide Web

Een jaar na de ontwikkeling van de HTTP kwam de grootste uitvinding in de geschiedenis van de communicatie: World Wide Web. Dit is een ruimte in het Internet waar documenten en toepassingen worden aangeboden. Deze zijn doormiddel van hyperlinks met elkaar gekoppeld. De World Wide Web is de grootste uitvinding ooit, omdat gebruikers nu via het internet kunnen communiceren en informatie vinden.

1993, 1996

HTML en CSS

Om structuur in de informatie te houden werd dat doormiddel van HTML gevisualiseerd. HTML staat voor Hypertext Markup Language en maakt gebruik van bepaalde elementen en tags, waardoor informatie in secties gestructureerd kan worden. Alleen was de opmaak nog erg basic. Een webpagina bestond uit een blanco pagina met tekst. Om dit aan te passen werd CSS toegepast (Cascading Style Sheets). Dit is een code om HTML documenten op een visuele manier op te maken. Denk hierbij aan de kleur, typografie en layout.

begin 21e eeuw

Fixed width en fluid layout

De desktopschermen van de meeste mensen hadden een breedte van 640 pixels. Designers maakten hun ontwerpen gebaseerd op deze vaste breedte, wat fixed width layout wordt genoemd. Dit deden ze vooral in Adobe Photoshop, omdat je daarin vaste afmetingen moet aangeven. Vervolgens groeiden de schermen naar 800 pixels en daarna 1024 pixels. Niet elke website was ontworpen voor de juiste breedte, dus moest hier een oplossing voor komen. Designers kwamen met de liquid layout, zodat gebruikers een website op verschillende formaten schermen kon bekijken.

2007

the iPhone

In 2007 kondigde Steve Jobs de iPhone aan. Dit werd de mobile device waar je op een klein browserscherm de World Wide Web op kon gebruiken. Gebruikers hadden alleen moeite om desktop websites op zoâ€™n kleine device te bekijken, waardoor de vraag naar flexible layout groter werd. Hierdoor kwamen designers met een oplossing: the mobile web. Met deze oplossing werd er een subdomein voor de mobiele website gemaakt: m.voorbeeld.com of mobile.voorbeeld.com. Nu werd het voor de gebruikers mogelijk op een gemakkelijke manier op hun iPhone door het internet te browsen

2009

Mobile First

Door het gebruik van the mobile web kon de gebruiker op de smartphone browsen. Op een gegeven moment kwam het idee om de layout voor een klein device anders te maken als die van een groter device: One Size Fits All. Vervolgens kwam Luk Wroblewski in 2009 met de term Mobile First. Met dit idee gaat het erom dat er weinig ruimte op de mobiele device is, waardoor je content moet prioriteren. Mobile First zorgde ervoor dat designers zich gingen focussen op de meeste belangrijke content.

2010

Responsive webdesign

In 2010 presenteerde Ethan Marcotte Responsive webdesign. Dit idee was gerelateerd aan het idee dat gebouwen kunnen veranderen en aanpassen aan de behoeftes van de mensen die de gebouwen gebruiken. Waarom zou dat ook niet met webdesign kunnen? 10 jaar eerder werd dit idee ook door John Allsop gepresenteerd, maar het werd door de gebruikers afgewezen. Nu is er dus een term voor bestaande termen.
